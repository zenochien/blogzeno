Đéo hiểu .Net Core là cái gì mà tui mù công nghệ

October, 4040 2-minute read

.Net Core is Open Source (Tạm dịch: .Net Core là nguồn mở)

This is a natural progression of our open source efforts, which already covers the managed compilers (C#, VB, and F#) as well as ASP.NET:

  • C# & Visual Basic (“Roslyn”)
  • Visual F# Tools
  • ASP.NET 5
  • Entity Framework

This takes it to the next level by extending it to the .NET runtime and the core framework.

  • .NET Core Framework

He ask ‘What is .NET Core?’

.NET Core is a modular development stack that is the foundation of all future .NET platforms It’s already used by ASP.NET 5 and .NET Native. I’ll go into more detail on what NET Core is and how it relates to the NET Framework in a following blog post.

Why do we open source .NET Core?

There are two big reasons why we decided to open source .NET Core:

  • Lay the foundation for a cross platform .NET (Đặt sự thiết lập cho một nền tảng chéo)
  • Build and leverage a stronger ecosystem (Xây dựng và tân dụng một hệ sinh thái mạnh mẽ)

Let me go into more detail.

As a .NET developer you were able to build & run code on more than just Windows for a while now, including Linux, MacOS, iOs and Android.

Our choice of using GitHub

So @ahejlsberg tweets TypeScript 5x faster compiler is on GitHub. I suspect causation. 
GitHub makes your code 5x faster. Anders says so.
Tạm dịch: Thế nên, @ahejlsberg tweets TypeScript 5x tốc độ nhanh hơn trên GitHub. 
Tôi nghi ngờ kết qua. GitHub mang code của bạn tốc độ gấp 5x. Anh ấy nói vậy

As a principle (principle = nguyên tắc), we don’t want to ask the community to come to where we are. Instead, we want to go where the community already is. Based on feedback that many other projects have received it seems the majority of the .NET community is on GitHub.

    You invoke build.cmd
